Create `src/components/TeachBackViewer.jsx` with the following structure:

**Imports:**
```javascript
import { useState, useRef, useCallback, useEffect } from "react"
import { motion } from "framer-motion"

```

**SpeechRecognition detection (top of file, outside component):**
```javascript
const SpeechRecognition = typeof window !== "undefined"
  ? window.SpeechRecognition || window.webkitSpeechRecognition
  : null
const hasSpeech = Boolean(SpeechRecognition)
```

**Phase state machine enum (per research Pattern 1):**
```javascript
const PHASES = {
  OFFLINE: "offline",
  IDLE: "idle",
  RECORDING: "recording",
  REVIEWING: "reviewing",
  EVALUATING: "evaluating",
  FEEDBACK: "feedback",
}
```

**Component: `export function TeachBackViewer({ moduleId, onComplete })`**

State:
- `phase` — initialized via `() => navigator.onLine ? PHASES.IDLE : PHASES.OFFLINE` (lazy initializer)
- `transcript` — string, default ""
- `evaluation` — null or `{ score, pass, correct, missed }`
- `error` — null or string
- `promptItem` — selected once on mount via `useState(() => { ... random selection from TEACHBACK_PROMPTS[moduleId] })`

Refs:
- `completedRef = useRef(false)` — guards onComplete
- `recognitionRef = useRef(null)` — holds SpeechRecognition instance

**Online/offline listener (per D-21, D-22):**
```javascript
useEffect(() => {
  const goOnline = () => setPhase(p => p === PHASES.OFFLINE ? PHASES.IDLE : p)
  const goOffline = () => setPhase(PHASES.OFFLINE)
  window.addEventListener("online", goOnline)
  window.addEventListener("offline", goOffline)
  return () => {
    window.removeEventListener("online", goOnline)
    window.removeEventListener("offline", goOffline)
  }
}, [])
```

**Voice handlers (per D-05, D-06, D-07, D-10):**

`handleStartRecording`:
- Create `new SpeechRecognition()`, set `continuous = true`, `interimResults = false`, `lang = "en-US"`
- On `onresult`: assemble transcript from `event.results`, call `setTranscript(text)` and `setPhase(PHASES.REVIEWING)`
- On `onerror`: set error message, set phase to IDLE
- Call `recognition.start()`, store in `recognitionRef.current`
- Set phase to RECORDING
- Add 60-second safety timeout: `setTimeout(() => recognitionRef.current?.stop(), 60000)`

`handleStopRecording`:
- Call `recognitionRef.current?.stop()`
- Phase transition happens in `onresult` callback

**Submit handler (per D-04, D-13, D-15, D-16, D-17):**

`handleSubmit`:
- Set phase to EVALUATING, clear error
- `fetch("/api/evaluate", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ moduleId, conceptArea: promptItem?.conceptArea, explanation: transcript }) })`
- On success (res.ok): parse JSON, set evaluation, set phase to FEEDBACK
- If `parsed.pass && !completedRef.current`: call `onComplete()`, set `completedRef.current = true`
- On failure: set error string "Couldn't reach the server -- check your connection and try again", set phase to REVIEWING

**Try Again handler (per D-17):**

`handleTryAgain`:
- Set phase to IDLE, clear transcript, clear evaluation, clear error
- Do NOT reset completedRef (per research anti-pattern note)

**Render logic — switch on `phase`:**

**OFFLINE phase:**
- Centered layout with WifiOff icon
- Text: "Needs internet connection" (per D-21, D-22)
- Muted styling with `var(--text-muted)`

**IDLE phase:**
- Show prompt text from `promptItem.prompt` at top (per D-18)
- Show concept area label: small muted text showing `promptItem.conceptArea`
- If `hasSpeech`: render mic button (circular, uses Mic icon from lucide)
- Always render textarea with placeholder "Type your explanation here..." (per D-11, D-12)
- If `hasSpeech`: textarea is secondary (smaller, below mic), with label "Or type instead"
- If `!hasSpeech`: textarea is primary (full width), no mic button shown
- Submit button disabled when transcript is empty

**RECORDING phase:**
- Show prompt text at top
- Pulsing mic button using `motion.button` with `animate={{ scale: [1, 1.15, 1], opacity: [1, 0.7, 1] }}` and `transition={{ duration: 1.2, repeat: Infinity, ease: "easeInOut" }}` (per D-08, research Pattern 5)
- MicOff icon inside the pulsing button (tap to stop)
- Text: "Listening... tap to stop"

**REVIEWING phase (per D-07):**
- Show prompt text at top
- Editable textarea pre-filled with transcript value
- `onChange` updates transcript state
- Submit button (Send icon + "Submit") enabled
- If `hasSpeech`: small "Re-record" button with Mic icon (tapping starts new recording per D-10, replaces transcript)
- Show error message if error state is set

**EVALUATING phase:**
- Show prompt text at top
- Loading indicator: "Evaluating your explanation..." with a subtle pulse animation
- No interactive elements

**FEEDBACK phase (per D-13, D-14, D-15, D-16, D-17):**
- Score display: large number `evaluation.score` out of 5, with visual treatment (e.g., colored circle or badge)
- Pass/fail indicator text: "Great job!" for pass, "Keep practicing!" for fail — encouraging tone per ADHD-aware design
- "What you got right:" section with bullet list from `evaluation.correct` array
- "What to work on:" section with bullet list from `evaluation.missed` array
- If `!evaluation.pass`: show "Try Again" button (RefreshCw icon) that calls `handleTryAgain` (per D-17)
- If `evaluation.pass`: show a brief success message (activity auto-completed)

**No-prompts guard:** If `promptItem` is null (empty prompt array), render: "No prompts available for this module."

**Styling approach:**
- Inline styles using CSS custom properties: `var(--text-primary)`, `var(--text-muted)`, `var(--bg-card)`, `var(--border)`, `var(--accent)`
- Tailwind utility classes for layout: `flex`, `flex-col`, `items-center`, `gap-3`, `w-full`, `text-center`
- Mic button: 56px circular, accent background, white icon
- Textarea: full width, 4 rows, rounded-lg, border using var(--border), bg using var(--bg-card)
- Submit button: accent background, white text, rounded-lg, px-4 py-2
- Score display: large font size (text-3xl or equivalent)
- Bullet lists: left-aligned, small text, appropriate spacing
