**Update vite.config.js (D-19, D-20, D-21):**

In `vite.config.js`, find the `workbox` section (currently lines 43-45):
```js
workbox: {
  globPatterns: ["**/*.{js,css,html,ico,png,svg,woff2}"],
},
```

Add `runtimeCaching` to it:
```js
workbox: {
  globPatterns: ["**/*.{js,css,html,ico,png,svg,woff2}"],
  runtimeCaching: [
    {
      urlPattern: /\/audio\/.+/,
      handler: "CacheFirst",
      options: {
        cacheName: "audio-cache",
        expiration: {
          maxEntries: 10,
          maxAgeSeconds: 30 * 24 * 60 * 60,
        },
      },
    },
  ],
},
```

Decision rationale:
- `CacheFirst` (D-19): Serves cached audio on repeat plays, falls back to network on first play. NetworkFirst would re-fetch on every play even when cached.
- `globPatterns` unchanged (D-20): Audio files (.mp3) are already excluded from precache. No change to globPatterns needed.
- `maxEntries: 10` (D-21): Covers all 8 modules with 2 spare entries. Prevents unbounded cache growth.
- `maxAgeSeconds: 30 * 24 * 60 * 60` (D-21): 30 days. Audio content is stable between releases.
